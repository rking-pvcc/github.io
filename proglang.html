Programming Languages Terminology

Terms Defintions
Machine Language Programs for modern computers consist of sequences of instructions that are encoded as numeric digits such an encoding system is known as the machine language.
Debugging Writing programs in a machine language is a tedious task that often leads to errors that must be located and corrected (a process known as debugging) before the job is finished.
Assembler Once the mnemonic system with identifiers was established, programs called assemblers were developed to convert mnemonic expressions into machine language instructions.
Assembly Language A mnemonic system for representing programs is collectively called an assembly language
Translator A translator was written that translated programs expressed in these high-level primitives into machine-language programs.
Compiler Such a translator was similar to the 2nd-generation assemblers, except that it often had to compile several machine instructions into short sequences to simulate the activity requested by a single high-level primitive. Thus, these translation programs were often called compilers.
Intrepreter An alternative to translators, called interpreters, emerged as another means of implementing third-generation languages. They executed the instructions as they were translated instead of recording the translated version for future use.
Cross-Platform Software Cross-platform software is software that is independent of an operating system's design as well as the machine's hardware design and is therefore executable throughout a network.
Natural Language Natural Languages are the languages that we use everyday such as English, Spanish, German, Latin, and so on..
Formal Language Formal Languages are programming languages used in softwares
Machine Independence The emergence of a third generation of programming languages that differed from previous generations in that their primitives were both higher level (in that they did not rely on the characteristics of a particular machine) and that they were machine independent (in that they did not rely on the characteristics of a particular machine).
High-Level Programming Language High-level programming languages include many kinds of abstractions, ranging from the simple - constants, literals, and variables - to moderate complexity - statements, expressions, and control structures - to the esoteric - procedural units, modules, and libraries.
Object-Oriented Programming (OOP) Another programming paradigm is the object-oriented paradigm, which is associated with the programming process called object-oriented programming (OOP). Following this paradigm, a software system is viewed as a collection of units, called objects, each of which is capable of performing the actions that are immediately related to itself as well as requesting actions of other objects.
Scripting Language A subset of the imperative programming languages is the collection of languages known as scripting languages which are typically used to perform administrative tasks rather than to develop complex programs.
Data Types: Integer The data type integer refers to numeric data consisting of whole numbers, probably stored using two's complement notation.
Data Types: Float The data type float (sometimes called real) refers to numeric data that might contain values other than whole numbers, probably stored in floating-point notation.
Data Types: String The data type string variable contains a collection of characters surrounded by double quotes used for storing text.
Primitive Data Type The data types that are included as primitives in a programming language, such as int for integer and char for character, are called primitive data types.
Data Structure In addition to date type, variables in a program are often associated with data structure, which is the conceptual shape or arrangement of data. One common data structure is the array, which is a block of elements of the same type such as a one-dimensional list, a two-dimensional table with rows and columns, or tables with higher dimensions.
Python Collection Data Structures: List Lists are used to store multiple items in a single variable and one of the 4 built-in data types in Python used to store collections of data.
Python Collection Data Structures: Tuple Tuples are used to store multiple items in a single variable and is a collection which is ordered and unchangeable.
Python Collection Data Structures: Set Sets are used to store multiple items in a single and are unordered, unchangeable, and unindexed. Set items are unchangeable, but you can remove and add new items
Python Collection Data Structures: Dictionary Dictionaries are used to store data values in key:value pairs and is a collection which is ordered, changeable, and do not allow duplicates.
Function A function, in its generic sense, is a set of instructions for performing a task that can be used as an abstract tool by other program units.
Local Variable As a general rule, a variable declared within a function is a local variable, meaning that it can be referenced only within that function.
Global Variable Variables whose scopes are not restricted to a particular part of a program are called global variables.
Source Program The process of converting a program from one language to another is called translation. The program in its original form is the source program.
Object Program The translated version is the object program.
